Data+ Health Networks and Disparities

Objective

During the summer of 2016, I researched under a program at the Information Initiative at Duke (iiD) called Data+. Data+ consisted of 25 projects for the summer, each with their own research objectives and mentors. Alongside Dr. Laura Sheble from the Duke Network Analysis Center, Dr. James Moody from the Department of Sociology and another undergraduate, I spent ten weeks modeling networks of comorbid disease. The purpose of the project was to model Electronic Health Record (EHR) patient data within a network to identify patterns of which diseases are often diagnosed together for what demographic, creating a predictive model for healthcare professionals to validate their diagnoses against.

Visit the website here.

Dataset

The data we were given was confidential, patient-protected healthcare data. All identifying information such as name, address and date of hospital visit were removed, and I had to complete HIPAA confidentiality modules before I was allowed access to the data. The entire dataset consisted of 1,048,576 paired entries taken from 2007-2011; all patients in the dataset had either a medical procedure or diagnosis on two or more occasions during the data collection period.

Entries were grouped into nodes (source, target). Each entry was coded under the International Classification of Diseases 9th edition (ICD-9).

  • Source, Target—ICD-9 Codes
  • Ethnicity: White, Black, Hispanic/Latino, Asian groups
  • Age: 0, 1, 2, 3, 4, 5, then ages 5+ were binned into 10 year brackets
  • Gender: M, F

Methods

  • JavaScript's D3: data visualization library to create basic node-edge list, with nodes representing ICD-9 codes and edges connecting earlier nodes to later nodes; each edge represents a patient with unique age, ethnicity and gender
  • Gephi: graph visualization software that produced static graph based on CSV spreadsheets
  • Sigma JS: final JavaScript library used to model a full node network
  • HTML/CSS: used to design website for global and ego network layout

We parsed data into CSV format organized by the parameters above. Then we loaded it into Gephi to produce a static graph, then used a Sigma JS plugin to translate the static graph into an interactive one. The ego-network shows closer connections between specific diseases, drawing the user's search query as a mySQL request and displaying the appropriate subset of the network.
